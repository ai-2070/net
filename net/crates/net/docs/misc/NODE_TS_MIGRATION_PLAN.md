# Node binding `.js` → `.ts` migration plan

**Status:** ✅ Executed 2026-05-06 on branch `node-ts-sdk` (commits `539ca2b0`, `9b3715c4`).

**Driver:** the hand-written Node binding modules (`errors.js`,
`mesh_rpc.js`) shipped with parallel `.d.ts` files that could
drift from the implementation. The `.d.ts` was the only guard on
the public type contract — and reviews of the nRPC work surfaced
several places where the two had quietly diverged (the
`RawMeshRpc` shape, the `breaker.armed` dead branch, the
`appError` helper signature). Migrating to a single TypeScript
source of truth catches that class of drift at build time.

## Scope

**In:** `errors.js` and `mesh_rpc.js` — the hand-written modules
that today ship alongside hand-written `.d.ts` declarations.

**Out:**
- `index.js` / `index.d.ts` — auto-generated by napi-rs from the
  Rust crate. Stays JS forever.
- Test files (`test/*.test.ts`) — already TypeScript.
- Other bindings (Python, Go) — N/A.
- ESM migration — separate decision, defer until a consumer asks.
- `src/` directory layout move — defer; flat layout works.

## Approach

Single source of truth: TypeScript source compiles to `.js` +
`.d.ts` that ship in the npm tarball. Module format stays CJS so
existing `require()` consumers keep working. Package layout stays
flat (no `src/` move).

## Steps

### 1. Wire up the TS pipeline

- Add `tsconfig.build.json`:
  - Extends the existing test-only `tsconfig.json`
  - `target: ES2022`, `module: CommonJS`, `moduleResolution: node`
  - `strict: true`, `declaration: true`, `skipLibCheck: true`
  - `outDir`/`rootDir` both `.` so import paths don't change
  - `noEmitOnError: true`
  - `include`: `["errors.ts", "mesh_rpc.ts"]`
  - `exclude`: `node_modules`, `dist`, `test`, `vitest.config.ts`,
    `index.js`, `index.d.ts`
- `package.json`:
  - `scripts.build:ts = "tsc -p tsconfig.build.json"`
  - `scripts.prepublishOnly = "npm run build:ts && napi prepublish -t npm"`
  - `scripts.typecheck = "tsc --noEmit -p tsconfig.build.json"`
    (CI-friendly noEmit alias)
- Root `.gitignore`: ignore compiled `errors.js`, `errors.d.ts`,
  `mesh_rpc.js`, `mesh_rpc.d.ts` (build artifacts; regenerated on
  publish via `prepublishOnly`)
- Verify `npm pack --dry-run` still produces a valid tarball

### 2. `errors.js` → `errors.ts`

- Convert verbatim — class shapes are nearly identical
- Pull types from existing `errors.d.ts` inline (`status?: number`,
  `direction?: 'encode' | 'delete'`, `elapsedMs?: number`)
- The `RpcCancelledError` added during the nRPC review work gets
  a real type rather than a hand-written `.d.ts` declaration
- Delete `errors.d.ts`
- Run vitest — the 53 tests should pass against the compiled
  output unchanged

### 3. `mesh_rpc.js` → `mesh_rpc.ts`

- Bigger file; types from `mesh_rpc.d.ts` come inline
  (`CallOptions`, `RawMeshRpc`, `RetryPolicyOptions`, etc.)
- Native binding shape is declared inline. The auto-generated
  `index.d.ts` shipped today was emitted from a napi build
  *without* the `cortex` feature, so it doesn't carry types for
  `MeshRpc` / `ServeHandle` / `RpcStream`. Released builds (which
  DO include cortex) provide these classes at runtime — a minimal
  structural shape declared here lets the TS source compile
  independently of which features the linked `.node` binary was
  built with.
- The `RawMeshRpc` interface added during the nRPC review becomes
  load-bearing — actual stubs have to conform. The test stubs
  (`StubRawMeshRpc`, `LoopbackHandlerRpc`, `CancelTrackingRaw`)
  may need to drop their `as unknown` escape hatches or grow a
  few methods.
- `wireAbortSignal` gets a real signature
- `appError(code: number, body: string | Buffer): Error` already
  typed; keep
- Delete `mesh_rpc.d.ts`
- Run vitest

### 4. Validate

- `npm run build:ts` from a wiped state regenerates all 4
  artifacts cleanly
- `npm run typecheck` is clean
- All 53 vitest tests still green
- `npm pack --dry-run` ships the expected 8 files
- All 4 compiled artifacts confirmed gitignored

## Risks + mitigations

- **napi-rs interaction**: `napi build` ignores `.ts` files; the
  TS step is independent. Already validated by the existing
  `.ts` test suite.
- **Test-stub conformance**: turning `RawMeshRpc` from
  documentation into a real type may surface stubs that lie
  about their shape. Fix as they appear (this is the win, not
  the cost).
- **CJS shape preservation**: pin `module: CommonJS` in tsconfig.
  Named `export class Foo` lowers to `exports.Foo = Foo` cleanly;
  consumers' `require('@ai2070/net/errors').CortexError` keeps
  working unchanged.
- **First publish after merge**: `npm pack --dry-run` in CI
  catches a misconfigured pipeline before it ships a broken
  package.
- **`index.d.ts` lacking nRPC types**: not a regression — that
  file is napi-generated from a feature-disabled build. Solved
  inline with structural interfaces in `mesh_rpc.ts`.

## Non-decisions (explicit)

- **ESM dual-build**: defer; separate decision driven by consumer
  demand
- **`src/` directory layout**: defer; flat layout works
- **Migrating `index.js`**: not possible (napi-generated); stays
  JS forever
- **Migrating Python/Go bindings**: N/A

## Effort actual

~90 minutes focused work: ~25min pipeline + `errors.ts`, ~50min
`mesh_rpc.ts` + structural-interface design, ~15min validation.
Two commits, each independently revertable.
