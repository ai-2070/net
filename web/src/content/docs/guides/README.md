# Guides

The pages in this section are task-oriented. Each one answers a "how do I…" question — how do I publish and consume events, how do I make a channel durable, how do I run a daemon that survives a node failure — and gives you the code, the model behind it, and the gotchas to know about.

Guides assume you've read the [Concepts](../concepts) section, or are willing to flip back when something doesn't make sense. They're meant to be read in any order; pick the one closest to what you're trying to do.

Everything here is grounded in real Net APIs. If you need the exhaustive signature of a method or the full grammar of a configuration object, the [Reference](../reference) section is where to look.
